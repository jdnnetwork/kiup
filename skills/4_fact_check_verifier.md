# 팩트체크 검증 스킬

## 목적
기관분석·업계분석 보고서에서 **출처가 달린 문장**의 사실 정확성을 검증하고, 오류가 있는 문장만 취소선+정정 형태로 보고한다.

## 검증 대상 판별 규칙

### 검증 대상 (출처가 있는 문장만)
- 문장 뒤에 `[출처명]`, `[출처명](URL)`, 또는 하단에 `(출처 : ...)` 형태로 출처가 명시된 문장
- 해당 문장 안의 **수치, 날짜, 고유명사, 제도명, 조직명, 비율, 금액** 등 객관적 사실

### 검증 제외
- `[생각]` 태그가 붙은 문장 (필자의 해석·의견)
- 출처가 없는 일반 서술문
- 하단 출처 블록 자체 (그대로 보존)

## 검증 절차 (5단계)

### 1단계: 문서 파싱 — 검증 대상 문장 추출
- 문서 전체를 읽고, 출처가 달린 문장을 모두 리스트업한다
- 각 문장에서 검증 가능한 **팩트 포인트**를 추출한다:
  - 수치 (금액, 인원, 비율, 증감률)
  - 날짜·시점 (설립일, 시행 연도, 기준 시점)
  - 고유명사 (기관명, 사업명, 제도명, 부서명)
  - 제도 구조 (부보율, 금리, 한도, 지원 대상)
  - 인용 발언 (누가, 무엇을)
  - **행위 주체** (누가 했는가 — 아래 '주체 귀속 오류' 섹션 참조)

### 2단계: URL 기반 검증 (URL이 있는 경우)
- 출처에 URL이 포함된 경우, `web_fetch`로 해당 페이지를 직접 방문하여 원문 대조
- 원문의 정확한 수치·표현과 문서의 기재 내용을 1:1 비교
- **특히 주의할 오류 유형:**
  - 수치 반올림·전사 오류 (5만1천 → 5만2천)
  - 연도 오기 (출처 작성 시점의 "올해"를 다른 연도로 기재)
  - 기관명·부서명 오기 (플랜트금융부 → 첨단산업금융부)
  - 소속 오류 (A본부 소속을 B본부 소속으로 기재)
  - 비율·증감률 계산 오류
  - **주체 귀속 오류** (아래 별도 섹션 참조)

### 3단계: 키워드 검증 (URL이 없는 경우)
- URL이 없는 출처(예: `[위키백과]`, `[한국경제]`, `[서울시]`)는 **출처명 + 핵심 키워드**로 웹 검색
- 검색 쿼리 설계 원칙:
  - 출처 매체명 + 문장의 핵심 수치/고유명사 조합
  - 예: `"한국경제 서울 개업률 8.1% 2025"`, `"서울시 중소기업육성자금 2조4000억"`
- 검색 결과에서 원문을 찾아 문서 기재 내용과 대조
- 원문을 찾을 수 없는 경우: 동일 주제의 다른 신뢰 출처 2개 이상으로 교차 검증

### 4단계: 공식 사이트 교차 검증
- 기관 조직도, 제도 설명 등은 **해당 기관 공식 홈페이지**를 직접 방문하여 확인
- 예: 조직 구조 → 공식 조직도 페이지 fetch
- 예: 제도 부보율 → 공식 제도개요 페이지 fetch
- 뉴스 기사 수치도 가능하면 공식 보도자료와 대조

### 5단계: 판정 기준
- **오류 확정**: 원문과 문서 기재 내용이 명백히 다른 경우
- **오류 미확정**: 원문을 찾을 수 없고, 교차 검증도 불가한 경우 → "검증 불가" 표기
- **정확**: 원문과 일치하거나, 의미상 동일한 경우 → 보고하지 않음 (정확한 건 언급 불필요)

## 출력 형식

### 오류가 있는 경우만 보고
```
**위치: [장/절 번호 및 소제목]**

> ~~[오류 문장 또는 오류 부분]~~ [출처명]

→ [출처 원문 확인 결과 정정 내용]
```

### 검증 불가인 경우
```
**위치: [장/절 번호 및 소제목]**

> [해당 문장] [출처명]

→ ⚠️ 검증 불가: [사유. 예: 해당 출처 원문을 찾을 수 없음]
```

### 모든 검증 완료 후 요약
```
---
검증 완료. 출처가 있는 문장 중 오류 **N건**, 검증 불가 **N건**입니다.
나머지 출처 문장은 모두 해당 출처 원문과 일치합니다.
```

## 검증 시 특별 주의 사항

### 주체 귀속 오류 (가장 빈번한 오류 유형)
기사는 문맥상 주어를 생략하거나 모호하게 쓰는 경우가 많다. 보고서 작성자가 이를 특정 기관의 행위로 단정하면 **주체 귀속 오류**가 발생한다. 이 오류는 수치 오류보다 발견하기 어렵지만, 면접·보고서에서 치명적일 수 있다.

**검증 방법:**
1. 문서에서 "A기관이 ~했다/~한다"는 문장을 만나면, 출처 원문에서 그 행위의 주체가 정말 A기관인지 확인
2. 원문에서 주체가 명시되지 않았거나 다른 주체인 경우 → 오류로 판정

**주요 오류 패턴 — 공공기관:**

| 패턴 | 예시 | 설명 |
|------|------|------|
| **상위기관 vs 하위기관** | "재단이 315억원을 투입" vs 실제 "서울시가 315억원을 투입하고 재단이 집행" | 예산 편성 주체(시)와 사업 수행 주체(재단)가 다름 |
| **정부 vs 공공기관** | "무역보험공사가 법을 개정" vs 실제 "정부(산업부)가 법 개정을 추진" | 법·제도 개정 주체는 정부이지 공공기관이 아님 |
| **협약 vs 단독** | "A기관이 지원한다" vs 실제 "A기관과 B기관이 협약을 맺고 공동 지원" | 복수 주체를 단일 주체로 축소 |
| **발표 vs 실행** | "A기관이 발표했다" vs 실제 "B장관이 회의에서 발표" | 발표 주체와 실행 기관이 다를 수 있음 |
| **기사 주어 생략** | 기사: "올해 275조원 무역보험 지원" → 보고서: "무역보험공사가 275조원을 지원" | 기사가 주어를 생략했는데 보고서가 임의로 주체를 지정 |
| **위탁 vs 직접** | "재단이 사업을 운영" vs 실제 "서울시가 설계하고 재단에 위탁" | 위탁받아 수행하는 것과 자체 기획·운영은 다름 |

**주요 오류 패턴 — 사기업:**

| 패턴 | 예시 | 설명 |
|------|------|------|
| **그룹 vs 계열사** | "삼성이 반도체 300조 투자" vs 실제 "삼성전자가 투자" | 그룹 전체와 특정 법인은 별개 주체. 매출·투자·수주 실적은 법인 단위로 귀속해야 정확 |
| **본사 vs 해외법인** | "현대차가 미국 공장 가동" vs 실제 "현대모터매뉴팩처링앨라배마(HMMA)가 가동" | 법적 주체가 다른 현지법인인 경우. 특히 소송·규제·고용 맥락에서 주체 구분 중요 |
| **단독 vs 합작법인(JV)** | "LG가 배터리 공장 건설" vs 실제 "LG엔솔과 GM의 합작법인 얼티엄셀즈가 건설" | 합작투자를 단독 투자로 기재하면 투자 규모·지분·의사결정 구조가 왜곡됨 |
| **대표이사 발언 vs 회사 결정** | "A사가 진출하겠다고 밝혔다" vs 실제 "CEO가 컨퍼런스에서 개인 의견으로 언급" | IR 공시·이사회 결의 vs 언론 인터뷰·행사 발언은 구속력이 다름 |
| **원청 vs 하청(협력사)** | "삼성물산이 시공" vs 실제 "삼성물산이 수주하고 협력사가 현장 시공" | 수주 주체와 실제 시공/제조 주체가 다를 수 있음 |
| **인수 전후 주체 혼동** | "A사가 2023년 매출 5조" vs 실제 인수 전이라 당시 B사 명의 실적 | M&A 전후로 법인명·실적 귀속이 바뀜. 인수 완료 시점 기준으로 주체 판단 |
| **사업부 vs 분사 법인** | "A사 배터리 사업 매출 30조" vs 실제 분사 후 별도 법인(A에너지솔루션) 실적 | 사업부 시절 실적과 분사 후 실적의 법적 주체가 다름 |
| **투자 유치 vs 자체 투자** | "A사가 1조원을 투자" vs 실제 "A사가 1조원 투자를 유치(받은 것)" | 투자한 것과 투자받은 것은 정반대 행위. 기사 헤드라인이 모호하게 쓰는 경우 많음 |

**판정 기준:**
- 원문에서 주체가 A인데 문서가 B로 기재 → **오류 확정**
- 원문에서 주체가 생략되어 있고, 문서가 특정 기관으로 단정 → 문맥상 합리적 추론이면 **허용**, 문맥으로도 불분명하면 **주의 표기** (⚠️)
- 원문에서 "A와 B가 공동으로"인데 문서가 "A가" → **오류 확정** (공동 주체를 단일 주체로 축소)
- 예산 편성 주체와 사업 집행 주체가 다른 경우: 문서가 어느 쪽 맥락인지에 따라 판단. "투입"은 예산 주체, "운영/수행"은 집행 주체에 귀속되는 것이 정확

### 연도 오기 함정
- 뉴스 기사에서 "올해", "지난해", "내년" 등 상대적 시점 표현 → 기사 작성일 기준으로 절대 연도 환산 후 문서 기재 연도와 대조
- 예: 2024년 7월 기사의 "올해 1분기" = 2024년 1분기 (문서가 2025년으로 기재하면 오류)

### 수치 전사 오류
- 천 단위 이하 숫자 차이도 오류로 판정 (5만1천 ≠ 5만2천)
- 비율 소수점 차이도 오류로 판정 (6.3% ≠ 6.5%)

### 조직·제도명 정확성
- 공식 명칭과 1글자라도 다르면 오류로 판정
- 소속(어느 본부/부서 산하인지)이 다르면 오류로 판정

### 출처 URL과 내용 불일치
- URL은 A 페이지인데 설명은 B 제도인 경우도 지적 대상

## 검증 순서 최적화

1. **먼저** 수치가 포함된 문장부터 검증 (오류 발생 확률 가장 높음)
2. **다음** 행위 주체가 명시된 문장 (주체 귀속 오류 — 빈번하지만 발견 어려움)
3. **다음** 조직명·제도명·부서명이 포함된 문장
4. **마지막** 제도 설명·절차 관련 문장 (공식 사이트와 대조)

## 작업 효율화 규칙

- 같은 출처(동일 URL 또는 동일 기사)에서 여러 문장이 인용된 경우, 해당 출처를 1번만 fetch하고 관련 문장을 일괄 검증
- 이전 대화에서 이미 검증한 출처는 재검증하지 않음 (단, 새 문서에서 해당 출처의 다른 수치를 인용한 경우는 추가 검증)
- 검색 쿼리는 1-6단어로 간결하게, 매체명을 포함

## 검증 강도 레벨

사용자가 별도 지정하지 않으면 **기본(Standard)** 적용:

| 레벨 | 설명 | 검증 범위 |
|------|------|-----------|
| Light | 수치·금액만 검증 | 숫자가 포함된 문장만 |
| **Standard** | 수치 + 고유명사 + 제도명 | 출처 있는 모든 사실 문장 |
| Deep | Standard + 모든 URL 직접 방문 + 교차검증 2회 이상 | 전수 조사 |

사용자가 "빡세게", "철저하게", "꼼꼼하게" 등의 표현을 쓰면 **Deep** 레벨 적용.

## ★★★ 자동화 환경 실행 규칙 (Routine 전용) ★★★

이 섹션은 자동화 환경(Claude Code Routine 등)에서 이 스킬이 실행될 때 적용되는 규칙이다.

### 인풋 자동 읽기

자동화 환경에서는 사용자에게 묻지 않고 다음 두 파일을 자동으로 검증 대상으로 삼는다:

1. **검증 대상 1**: `output/{업계명}_업계특성.docx` (1번 스킬 결과물)
2. **검증 대상 2**: `output/{기업명}_경쟁력관점_보강.docx` (3번 스킬 결과물)

기업명·업계명은 `inputs/기업명.txt`, `inputs/업계명.txt`에서 자동으로 읽어 경로를 구성한다.

**파일이 없으면 즉시 중단:** 두 파일 중 하나라도 없으면 작업을 진행하지 않고 어떤 파일이 없는지 출력한다.

### ★ 검증 범위 — 간결 원칙 (CRITICAL) ★

**오류로 의심되는 부분만 보고한다. 정제본·정정본은 절대 만들지 않는다.**

- ❌ `{기업명}_경쟁력관점_정제.docx` 같은 정제본 docx 생성 금지 (토큰 폭증 방지)
- ❌ 원본 보고서를 다시 작성하거나 재구성하지 않음
- ✅ 오류 위치와 정정 내용만 간단히 리포트 형식으로 출력

### 검증 강도

자동화 환경에서는 기본 **Standard** 레벨을 적용한다 (수치 + 고유명사 + 제도명).

검증 시간이 오래 걸려 Routine이 타임아웃될 우려가 있으면, 다음 우선순위로 검증한다:
1. 수치·금액·날짜 (오류 발생률 가장 높음)
2. 행위 주체 (주체 귀속 오류)
3. 조직명·제도명·부서명

위 3개를 모두 검증한 뒤 시간이 남으면 나머지 사실 문장도 검증한다.

### 아웃풋 저장 위치

- 경로: `output/{기업명}_팩트체크리포트.docx`
- 예: `output/산은캐피탈_팩트체크리포트.docx`

**리포트 구조 (간결):**

```
# 팩트체크 리포트 — {기업명}

검증 대상:
- output/{업계명}_업계특성.docx
- output/{기업명}_경쟁력관점_보강.docx

검증 강도: Standard

---

## 1. {업계명}_업계특성.docx

### 오류 확정 (N건)

**위치: [장/절 번호 및 소제목]**
> ~~[오류 문장 또는 오류 부분]~~ [출처명]
→ [출처 원문 확인 결과 정정 내용]

(반복)

### 검증 불가 (N건)

(있는 경우만 표기)

---

## 2. {기업명}_경쟁력관점_보강.docx

(동일 형식)

---

## 종합 요약

- 1번 보고서: 오류 N건, 검증 불가 N건
- 2번 보고서: 오류 N건, 검증 불가 N건
- 5번 스킬(지원동기·포부)에서 사용 금지할 정보 리스트:
  - [오류 1: 어떤 사실/수치/주체가 잘못되었는지 한 줄 요약]
  - [오류 2: ...]
  - (이하 모든 "오류 확정" 항목을 5번 스킬이 참조하기 쉽게 요약)
```

위 "종합 요약"의 **사용 금지 정보 리스트**가 핵심이다. 5번 스킬이 이 리스트를 참고해서 해당 정보를 사용하지 않는다.
